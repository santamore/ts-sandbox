# Учебная дорожка: от основ к TypeScript

Отмечай выполненные темы: замени `[ ]` на `[x]` в своём редакторе.

---

## 1. Самые основы программирования

- [x] Что такое программа, исходный код, выполнение (runtime)
- [x] Комментарии в коде: зачем и как писать
- [x] Вывод в консоль (`console.log`) — как смотреть результат

## 2. Переменные и константы

- [x] Что такое **переменная** (имя + значение, значение можно менять)
- [x] Что такое **константа** (имя + значение, переприсваивать нельзя)
- [x] `const` — по умолчанию для значений, которые не должны «переезжать» на другое
- [x] `let` — когда значение должно обновляться
- [x] Почему не стоит использовать устаревший `var` (или: в чём отличие от `let`)

## 3. Примитивные типы данных (идея + в JS/TS)

- [x] **Число** (`number`): целые и дробные, арифметика
- [x] **Строка** (`string`): текст в кавычках, склеивание строк
- [x] **Логический тип** (`boolean`): `true` / `false`
- [x] **`undefined`** — значение «не задано»
- [x] **`null`** — намеренное «пусто» (идея отличия от `undefined`)
- [x] **BigInt** (по желанию) — очень большие целые числа
- [x] **Symbol** (позже, по желанию) — уникальные идентификаторы

## 4. Операторы и выражения

- [x] Арифметика: `+`, `-`, `*`, `/`, `%`, степень `**`
- [x] Сравнение: `===`, `!==`, `<`, `>`, `<=`, `>=`
- [x] Логика: `&&`, `||`, `!`
- [x] Сокращённые формы: `+=`, `-=`, инкремент/декремент

## 5. Управление потоком

- [x] Условия: `if`, `else`, `else if`
- [x] Тернарный оператор `условие ? да : нет` (краткая форма)
- [x] `switch` (когда много вариантов одного значения)
- [x] Циклы: `while`, `for`
- [x] Цикл по коллекции: `for...of` (для массивов и итерируемого)

## 6. Функции

- [x] Объявление функции: `function имя() { }`
- [x] Параметры и возвращаемое значение (`return`)
- [x] Область видимости: где видна переменная
- [x] Стрелочные функции: `() => { }` — когда удобны
- [x] Чистые функции (идея): без скрытых побочных эффектов

## 7. Массивы и объекты

- [x] Массив: создание, индекс `arr[0]`, длина `length`
- [x] Основные методы: `push`, `pop`, `map`, `filter` (хотя бы первые два сразу, остальные чуть позже)
- [ ] Объект: пары «ключ — значение», доступ через `.` и `[]`
- [ ] Деструктуризация объектов и массивов (по мере готовности)

## 8. Строки и шаблоны (современный JS)

- [ ] Шаблонные строки: `` `Привет, ${имя}` ``
- [ ] Полезные методы строк: `includes`, `slice`, `trim` (по мере задач)

## 9. Ошибки и отладка

- [ ] Чтение сообщения об ошибке (строка, номер)
- [ ] `try / catch` — перехват ожидаемых сбоев
- [ ] `throw` — явное создание ошибки

## 10. Модули

- [ ] Идея: разбить код на файлы
- [ ] `export` и `import` (именованный и `default`)

## 11. Асинхронность (после уверенного синхронного кода)

- [ ] Зачем нужны асинхронные операции (сеть, таймеры)
- [ ] `Promise`: состояния, `then`, `catch`
- [ ] `async` / `await` — читаемый стиль работы с промисами

## 12. Введение в TypeScript

- [ ] Зачем типы: подсказки редактора и меньше ошибок до запуска
- [ ] Аннотации: `let x: number`, `function f(a: string): boolean`
- [ ] `interface` и `type` для описания объектов
- [ ] Union: `string | number`, опциональные поля `?`
- [ ] `readonly`, кортежи (базово)
- [ ] Generics: идея «тип-параметр» (`<T>`)
- [ ] `unknown` вместо `any`, сужение типов (`typeof`, проверки)
- [ ] Настройки `tsconfig`: `strict`, `noImplicitAny` (что включают)

## 13. Практика в проекте

- [ ] Запуск своего `.ts` через `tsc` или скрипт из `package.json`
- [ ] Маленькая задача своими руками: функции + типы + один модуль

1. Иди **сверху вниз**: следующая тема опирается на предыдущие.
2. После блоков **6–7** можно параллельно решать маленькие задачи в `src/fundamentals/`, если они у тебя есть в репозитории.
